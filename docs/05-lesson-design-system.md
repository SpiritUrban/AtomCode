# AtomCode Lesson Design System

> **Version:** 1.0
> **Applies to:** JSAtoms, CSSAtoms, HTMLAtoms, GitAtoms, ReactAtoms, TSAtoms та всі майбутні курси.

---

# 1. Philosophy

AtomCode — це не звичайний курс.

Це бібліотека атомарних знань.

Кожен урок пояснює лише **одну нову ідею**.

Після проходження уроку студент повинен зрозуміти одну концепцію, а не цілий розділ.

Великі теми будуються поступово з маленьких атомів.

---

# 2. Atomic Principle

## Один урок = одна нова ідея

Правильно:

* Що таке Variable
* Що таке String
* Що таке Number

Неправильно:

* Variables + let + const + scope
* Arrays + Objects + Functions

Кожен урок повинен відповідати лише на одне питання.

Приклад:

```
Що таке змінна?
```

або

```
Що таке Boolean?
```

---

# 3. Learning Formula

Кожен урок обов'язково проходить однаковий шлях.

```
Real Life

↓

Problem

↓

JavaScript Idea

↓

Code Example

↓

Rule
```

Це головний принцип AtomCode.

---

# 4. Storytelling

Кожен урок — маленька історія.

Вона повинна мати логіку.

## 1. Real Life

Починаємо зі знайомої ситуації.

Приклади:

* коробка
* двері
* гроші
* телефон
* книга
* список покупок
* цінник
* світло
* ключ

Не починаємо з програмування.

---

## 2. Problem

Герой стикається із проблемою.

Наприклад:

"Хочу запам'ятати ім'я."

---

## 3. JavaScript Idea

Показуємо, як JavaScript вирішує проблему.

Наприклад:

Variable.

---

## 4. Code Example

Мінімальний робочий приклад.

Без зайвого коду.

---

## 5. Rule

Одне речення.

Саме його людина повинна запам'ятати.

---

# 5. Lesson Layout

Кожна картинка має однакову структуру.

```
┌─────────────────────────────┐
│ Header                      │
├─────────────────────────────┤
│ Panel 1 │ Panel 2           │
├─────────┼───────────────────┤
│ Panel 3 │ Panel 4           │
├─────────────────────────────┤
│ Bottom Rule                 │
└─────────────────────────────┘
```

---

# 6. Header

Верх картинки містить:

```
Title

Subtitle
```

Приклад:

```
Variables

Boxes for your data
```

---

# 7. Panel Rules

Кожна панель повинна пояснювати лише одну думку.

Максимум:

* одна дія
* одна емоція
* одна ідея

Не перевантажувати.

---

# 8. The Four Panels

## Panel 1

Реальна ситуація.

Наприклад:

```
Хочу зберегти ім'я.
```

---

## Panel 2

Проблема або розвиток.

```
Як його отримати пізніше?
```

---

## Panel 3

Момент розуміння.

```
О!

Variable пам'ятає значення.
```

Саме тут народжується нова ідея.

---

## Panel 4

Код.

Мінімальний.

Правильний.

Зелена галочка.

---

# 9. Bottom Rule

Внизу завжди одне правило.

Приклади:

```
Variables store data.
```

```
Strings store text.
```

```
Booleans store true or false.
```

```
Numbers store numeric values.
```

Bottom Rule повинно бути коротким.

Бажано:

до 8 слів.

---

# 10. Character

У всіх уроках використовується один і той самий персонаж.

Він:

* навчається
* дивується
* помиляється
* знаходить рішення

Студент асоціює себе саме з ним.

---

# 11. Visual Style

Стиль:

Comic + Infographic.

Використовувати:

* великі заголовки
* яскраві кольори
* прості форми
* багато білого простору
* великі іконки
* мінімум тексту

Не використовувати:

* великі абзаци
* дрібний текст
* перевантажені схеми

---

# 12. Text Rules

Текст максимально простий.

Не писати мовою документації.

Писати мовою людини.

Добре:

```
Use quotes.
```

Погано:

```
Strings are immutable sequences of UTF-16 code units...
```

---

# 13. Code Rules

Код повинен:

* працювати
* бути мінімальним
* бути читабельним

Бажано:

3–5 рядків.

Максимум:

8 рядків.

---

# 14. Real Life First

Будь-яка концепція повинна починатися з реального світу.

Наприклад:

Variables

→ коробка

String

→ ім'я

Boolean

→ двері

Array

→ кошик покупок

Object

→ паспорт

Function

→ кавовий автомат

Promise

→ доставка піци

Event

→ натискання кнопки

DOM

→ конструктор LEGO

---

# 15. Difficulty Progression

Нові поняття вводяться лише після попередніх.

Наприклад:

```
Variables

↓

Primitive Types

↓

Number

↓

String

↓

Boolean

↓

null

↓

undefined

↓

typeof
```

Не можна пояснювати Promise до Functions.

Не можна пояснювати Objects до Variables.

---

# 16. Lesson Checklist

Перед додаванням уроку перевірити:

□ пояснюється одна нова ідея

□ є реальна аналогія

□ є проблема

□ є рішення

□ є код

□ є Bottom Rule

□ код короткий

□ текст простий

□ картинка не перевантажена

□ немає другої нової теми

---

# 18. Final Principle

AtomCode не навчає великими лекціями.

AtomCode будує знання атом за атомом.

Якщо після уроку людина може коротко пояснити одну нову концепцію своїми словами — урок створений правильно.

Кожен атом повинен бути настільки простим, щоб його можна було зрозуміти за 2–3 хвилини та запам'ятати надовго.

> **One lesson. One idea. One image. One memory.**
