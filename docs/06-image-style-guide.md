# AtomCode Image Style Guide

> **Version:** 1.0
> **Purpose:** Define a single visual language for every AtomCode lesson.

---

# 1. Purpose

Усі ілюстрації AtomCode повинні виглядати так, ніби вони створені одним художником в один день.

Користувач повинен миттєво впізнавати стиль незалежно від теми уроку.

Метою цього документа є забезпечення абсолютної візуальної єдності всіх курсів.

---

# 2. Master Template

Перший урок JS-001 **Variables** є офіційним Master Template.

Він є головним референсом для всіх майбутніх ілюстрацій.

При створенні нових уроків потрібно максимально повторювати:

* композицію
* пропорції
* кольори
* персонажів
* типографіку
* декоративні елементи
* стиль малювання
* стиль тіней
* стиль рамок

Змінюється лише зміст уроку.

Дизайн не змінюється.

---

# 3. Overall Style

Стиль можна описати як:

Comic Education

*

Modern Infographic

*

Bright Cartoon

*

Gaming UI

Враження повинно бути таким:

"Це навчальна картка з дуже дорогої сучасної освітньої платформи."

---

# 4. Canvas

Формат:

```text
1 : 1
```

Усі картинки квадратні.

---

# 5. Global Layout

Кожна картинка складається з однакових блоків.

```text
┌───────────────────────────────┐
│ Header                        │
├───────────────────────────────┤
│ Panel 1 │ Panel 2             │
├─────────┼─────────────────────┤
│ Panel 3 │ Panel 4             │
├───────────────────────────────┤
│ Bottom Rule                   │
└───────────────────────────────┘
```

Жодних інших варіантів композиції.

---

# 6. Header

Header займає приблизно 20% висоти.

Містить:

* жовтий JS mascot
* головний заголовок
* підзаголовок
* декоративний елемент справа

Композиція повинна залишатися незмінною.

---

# 7. Mascot

У верхньому лівому куті завжди знаходиться жовтий JS-персонаж.

Він є частиною бренду AtomCode.

Він не змінюється між уроками.

---

# 8. Lesson Character

Головний герой — один і той самий хлопець.

Вимоги:

* однаковий стиль волосся
* однакове обличчя
* однакові пропорції
* однаковий зелений худі
* однаковий стиль очей
* однаковий стиль емоцій

Не використовувати інших головних персонажів.

---

# 9. Panel Design

Усі панелі:

* однаковий розмір
* товста чорна рамка
* легке округлення
* окремий кольоровий фон

Фони:

Panel 1

теплий

Panel 2

холодний

Panel 3

фіолетовий

Panel 4

зелений

---

# 10. Number Badges

У кожної панелі є великий жовтий номер.

Стиль номерів однаковий.

Не змінювати форму.

---

# 11. Speech Bubbles

Використовувати класичні коміксні хмаринки.

Текст великий.

Добре читається навіть із маленького прев'ю.

---

# 12. Code Windows

Код показувати лише у стилізованому темному вікні.

Вікно повинно нагадувати редактор коду.

Обов'язково:

* темний фон
* кольоровий синтаксис
* мінімум рядків

---

# 13. Colors

Основна палітра:

Dark Blue

Yellow

White

Green

Purple

Blue

Orange

Чорний використовується лише для контурів.

Кольори повинні бути дуже насиченими.

---

# 14. Typography

Усі заголовки:

* жирні
* великі
* контрастні

Підзаголовки:

* менші
* прості
* легко читаються

Не використовувати тонкі шрифти.

---

# 15. Bottom Rule

Унизу завжди є темна горизонтальна смуга.

Ліворуч:

маленька декоративна іконка.

Посередині:

головне правило уроку.

Наприклад:

Variables store data.

Strings store text.

Booleans store true or false.

---

# 16. Visual Complexity

Кожна панель повинна містити лише один головний об'єкт.

Не перевантажувати деталями.

Погано:

кімната, люди, меблі, комп'ютер, телефон, автомобіль.

Добре:

коробка.

або

двері.

або

цінник.

---

---

# 19. Consistency

Новий урок не повинен виглядати як новий дизайн.

Він повинен виглядати як ще одна сторінка тієї самої книги.

Якщо покласти поруч:

JS-001

JS-087

JS-143

повинно створюватися враження, що вони намальовані одним художником.

---

# 20. AI Generation Rules

При генерації зображень завжди використовувати перший урок як референс.

Інструкція для AI:

* максимально повторити композицію
* максимально повторити кольори
* максимально повторити стиль персонажів
* максимально повторити типографіку
* максимально повторити стиль панелей
* змінити лише тему уроку

Не вигадувати новий дизайн.

Не змінювати композицію.

Не експериментувати зі стилем.

---

# 21. Final Principle

Стиль AtomCode — це не набір красивих картинок.

Це впізнаваний бренд.

Користувач повинен впізнати урок AtomCode ще до того, як прочитає його назву.

Якщо новий урок неможливо відрізнити від JS-001 за стилем — дизайн виконаний правильно.
